# MAGDALENE Context

## What It Is

MAGDALENE is a music-video construction program. It is built around a node graph
and an audio timeline, but its real subject is memory, witness, archive, rhythm,
image damage, and generated edits.

The user should be able to load a song, images, video fragments, lyrics, and
text, then build a system that turns those materials into a finished visual
piece. MAGDALENE should make programmed edits feel physical: beats become
cuts, stems become pressure, lyrics become screen events, images become
particles, and sections become shifts in composition.

## Relationship To FOUND / FOOTAGE

FOUND / FOOTAGE is the proof. It is an audio-led image rupture engine: a track
plus an image archive becomes a datamosh-warped, point-cloud-driven music
video. It already has important machinery:

- image/archive scanning
- audio analysis
- particle-based image morphing
- cell layouts
- music-reactive choreography
- post-process degradation
- live recording
- offline MP4 render

MAGDALENE is not a rename of FOUND / FOOTAGE. It is the editor that grows out
of what FOUND / FOOTAGE proved. FOUND / FOOTAGE can remain a finished art
engine. MAGDALENE turns the idea into a broader creative tool where the user
can see, edit, route, and save the systems that produce the video.

## Why A Separate Repo

MAGDALENE needs a different architecture and a different promise. FOUND /
FOOTAGE is an auto-maker with a strong identity. MAGDALENE is a program for
making many kinds of audio-reactive edits.

Keeping it separate avoids turning FOUND / FOOTAGE into a half-editor,
half-artwork. It also lets MAGDALENE make clean product decisions from the
start: project files, graph runtime, timeline, media library, inspector, and
export as first-class systems.

## Differentiation

MAGDALENE should not try to beat TouchDesigner at being TouchDesigner.
TouchDesigner is a general-purpose visual programming environment. MAGDALENE
should be narrower and sharper: a node-based editor that understands the shape
of music videos.

MAGDALENE differs from:

- **TouchDesigner**: not a low-level operator sandbox first; it has built-in
  concepts for songs, beats, stems, lyrics, cuts, archive media, typography,
  layouts, and export.
- **After Effects**: not layer/keyframe compositing first; graph logic and
  audio-derived structure are central.
- **Resolume/VJ tools**: not live clip triggering first; the first goal is
  authored exportable videos, with live performance later.
- **Generic music visualizers**: not presets reacting to frequency bands; users
  can build edit systems that understand moments, lyrics, sections, and media
  choice.

## Product Thesis

Most visualizers react to sound. MAGDALENE should edit with sound.

The program should expose music as structure:

- kick, snare, hats, bass, vocals, and other stems
- beat grid and bars
- sections such as intro, verse, build, drop, breakdown, and outro
- lyric moments
- manual markers and user-authored events

Those structures should be routable into visual decisions:

- when to cut
- what media to choose
- which image or video becomes dominant
- which layout appears
- how particles move
- when text appears
- how much degradation is applied
- when the frame becomes readable or collapses

## Tone

MAGDALENE should feel like serious creative software, but not sterile. It can
carry the ritual, archive, witness, and damage language from FOUND / FOOTAGE
without becoming a parody of darkness.

The interface should be usable first. The atmosphere should come from the
materials, naming, typography, and render output, not from making the tool hard
to use.

## Naming

MAGDALENE is the public project name. It carries art-history, witness, grief,
devotion, body, ritual, and image-making associations without being tied to one
single painting. It fits the idea of a tool that turns fragments into a charged
visual artifact.
